Introduction

Version 1.1

This version of the template adds the following features:

  • Easy template-wide color customization

  • Dark template color preset using the new color customization feature

Color Customization

This new feature allows you to easily edit the colors of the template by making use of SASS variables. We created a new SASS file (sass/_color-theme.scss) which centralizes the color variables of the template (you should not modify this file).
There is another file we created (sass/_color-theme-custom.scss) which is the one you will use to enter your custom colors by using sass/_color-theme.scss as a base.

Note: this feature allows you to change CSS based styles, to change colors of javascript plugin elements (like the hexagons, progress bars or the charts) you need to edit the corresponding javascript file. For example if you want to modify the colors of the user avatar hexagons, edit the js/global/user-avatar.hexagons.js file.
By doing this, you can achieve the same amount of customization as showed in the included Dark template version

To better illustrate how color customization should be done we describe the following usage example:

How to customize primary template colors.

  • Open sass/_color-theme.scss and copy the PRIMARY COLORS section

  • Paste the PRIMARY COLORS section in the sass/_color-theme-custom.scss file

  • Modify the variable values with your desired new colors. In general color variable names are self explanatory, $color-primary-background is used when there's text inside that color, for example on buttons and tags. If you don't know what a color variable references just try changing it with a hard to miss color and check what changed.

  • For the changes to reflect on the styles, you need to compile the SASS files by using sass/main.scss as entry point and css/styles.min.css as exit point (if you are using the included npm + Grunt version then everything will be compiled automatically by just saving the file).

  • After compiling and generating the new css/styles.min.css, you can visualize the changes on the browser by opening any HTML page of the template (You may need to do a force refresh - Ctrl + F5 if a cached version of the page is being loaded).

You can follow the same process to customize any of the other colors present in the sass/_color-theme.scss file.

Note: We don't recommend directly replacing the color variables in the sass/_color-theme.scss file. This file can be used as fallback if you want to reset changes you've made to color variables, if you do this directly on the file you may loose track of the changes. Also, you can change between default colors and your custom colors quickly by just commenting or deleting the @import 'color-theme-custom'; import statement on the sass/main.scss file.

How to Customize

Main Version

The files of the template can be found inside the HTML Template directory. This is the main version of the template that you can integrate with your desired development flow. This includes all the source files (HTML, SASS, CSS, JS, etc) that you can use to customize the template.

Optional Version (npm & Grunt)

This version is based on the use of npm as a package manager and grunt as a task runner.
First you will need to install the following:

  • Node

  • npm

Once you have installed them and verified that they are functioning correctly, navigate to HTML Template - npm and Grunt. Here you will find a file called package.json, this file contains the development and production dependencies of the template (including grunt, the task runner that we will be using), this is what is going to be managed by npm.

npm

In order to install these dependencies, navigate to HTML Template - npm and Grunt if you haven't already, and run the command npm install in your terminal.
This will take some minutes as it downloads and installs all the dependencies and related packages that they need. When it's done, you'll notice that a folder called node_modules will be created in the same directory, this folder has all the dependencies that were downloaded, make sure that you don't delete it or you'll have to run npm install again.

Now that we have everything setup, we only need to use grunt for development (note that you don't need to run npm install again, unless you add new dependencies or want to update them or npm itself).

Grunt

In the same directory (HTML Template - npm and Grunt) you will find a file called Gruntfile.js, this file contains the tasks that will be run each time you execute the grunt command on that same directory and when you make changes to the files of the template with the command running. So you only need to execute it once when you want to start customizing the template, and further changes to files in the src/ directory will trigger an automatic update and refresh of the files (livereload).

To start customizing the template, navigate to HTML Template - npm and Grunt and run grunt on your terminal. You'll notice that it will do some work and it should automatically open a tab on your browser, if it doesn't, open a new tab on your browser manually and enter localhost:8123 as the URL (that port number must not be in use by other application for it to work). If you need or want to change the port you can do so by editing the Gruntfile.js file in the connect:build section (line 76), you will also find the livereload port number there.

When executing the grunt command and/or when updating files on the src/ directory, the task runner will run the respective files through the configured compilation tools automatically. So, for example, if you want to edit styles, find the corresponding file in the src/sass/ directory, or create a new one and edit the styles, grunt will automatically compile the .scss files into one .css file, and update the browser tabs using livereload, so you can check the changes instantly.

The process is the same for adding or modifying images, HTML or Javascript files, Javascript files are transpiled using Babel, bundled using Browserify and minified with UglifyJS into one bundle that is included in all the HTML pages (app.bundle.min.js). The entry point for the Javascript bundle is the file src/js/app.js which is used to require all the files that will be put in the bundle

Editing Styles

The src/sass/main.scss file is the entry point for the compilation. If you want to add new style files, you need to include them here. If you want to edit existing styles, just edit the corresponding file in the src/sass/ directory and save the file, Grunt will compile the styles and refresh the page automatically.

Editing Scripts

The src/js/app.js file is the entry point for the compilation. If you want to add new script files, you need to include them here. If you want to edit existing scripts, just edit the corresponding file in the src/js/ directory and save the file, Grunt will compile the scripts and refresh the page automatically.

Google Maps

The template includes a google map widget. You can find the configuration file associated to it in js/global/maps.js. You can edit this file to input your desired map configuration.

Google API Key

For the map to work, you need to get an API Key from Google here and replace it on the script tags where it says key=YOUR_API_KEY. The callback argument must point to a function that will be used to create the map.

You will find those script tags on the HTML pages that have a map widget.

Twitter Widget

The template includes a Twitter widget. This widget is used to load the last 'n' (default: 3) tweets of your Twitter timeline.

In order for the widget to work you need to:

  • Use a PHP server (i.e. apache), since this widget uses PHP to communicate with the Twitter server.

  • Add your Twitter access tokens

  • Enter the screen name (screen_name) of the account you want to display tweets of, and (optionally) modify the amount of tweets to retrieve/display (count)

Add your Twitter access tokens

Open the PHP file includes/getTweets.php and add your credentials in the $ACCESS_KEYS array.

Add your Twitter access tokens

Open the Javascript file js/widget/twitter.js and edit the screen_name and count GET parameters with your desired values.
